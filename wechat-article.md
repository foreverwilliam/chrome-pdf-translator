# 我做了个 Chrome 插件，看英文 PDF 再也不用来回切翻译了

看英文论文、技术文档的时候，最烦的不是看不懂，是来回切。

复制一段，粘到翻译网站，看完再回去找位置。一篇 20 页的 PDF 下来，时间全花在切窗口上了。

所以我做了个 Chrome 扩展：**PDF Translator**。

拖一个 PDF 进去，左边显示原文，右边实时出中文翻译。不用切窗口，不用复制粘贴，看完一页翻一页。

---

## 它能做什么

**左右分栏对照阅读**，左边是 PDF 原文渲染，右边是逐段翻译结果。点击右边某段译文，还能展开看英文原文对照。

核心功能一览：

- ✅ 左右分栏对照，中间分隔线可拖动调整比例
- ✅ 翻译单页 or 一键翻译全部页面
- ✅ 逐段翻译，带实时加载动画
- ✅ 左右面板同步滚动
- ✅ PDF 翻页、缩放、拖放文件加载
- ✅ 暗色 / 亮色主题自由切换
- ✅ 双语对照模式

---

## 支持 4 种翻译服务

**DeepL**
翻译质量公认最好，免费版每月 50 万字额度，够用。

**OpenAI（GPT-4o）**
灵活强大，翻译学术论文效果很好。

**DeepSeek**
高性价比之选，V3 和 R1 两个模型可选。

**自定义 API**
兼容 OpenAI 格式的任意服务都能接，比如 Qwen、GLM、MiniMax、Ollama 本地模型等。只需填一个 base URL（比如 `https://your-api.com`），插件自动拼接完整路径，不用记 `/v1/chat/completions` 这种东西。

---

## 性能：一次请求翻一整页

最新版做了一个关键优化：

之前是逐段翻译，一页 20 段就要发 20 次 API 请求，串行等待，慢得要死。

现在改成整页合并为一次请求，20 段变 1 次调用。速度快了一个量级，API 费用也省了（system prompt 不再重复发送 20 遍）。

DeepL 更进一步，直接用它原生的数组批量接口，一次传所有段落，效率拉满。

## 3 分钟装好开始用

**第一步：安装扩展**

Chrome 地址栏输入 `chrome://extensions/`，右上角打开「开发者模式」，点「加载已解压的扩展程序」，选择下载好的项目文件夹。

**第二步：配置 API Key**

点击工具栏的插件图标，选择翻译服务，填入 API Key，保存。

**第三步：开始翻译**

点「打开翻译器」，把 PDF 拖进去，点翻译按钮，右侧面板实时出结果。

就这么简单。
